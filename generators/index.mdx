OpenAPI Generator can produce client SDKs, server stubs, documentation, and schema artifacts from a single OpenAPI specification. Each target language or framework is handled by a dedicated **generator** identified by its generator name.

## Generator categories

<CardGroup cols={2}>
  <Card title="Client generators" icon="code">
    Generate fully-typed HTTP client SDKs in 50+ languages. Clients handle authentication, serialization, and request lifecycle out of the box.
  </Card>
  <Card title="Server generators" icon="server">
    Generate server stubs and framework scaffolding. Implement the generated interfaces to wire your business logic with the correct contract.
  </Card>
  <Card title="Documentation generators" icon="book-open">
    Render your OpenAPI spec as HTML, AsciiDoc, Markdown, PlantUML diagrams, and more — without writing a line of markup by hand.
  </Card>
  <Card title="Schema generators" icon="database">
    Transform your OpenAPI spec into alternative schema formats: Avro, GraphQL, MySQL, PostgreSQL, Protobuf, Postman collections, and others.
  </Card>
</CardGroup>

## Stability levels

Every generator declares a stability level that reflects how production-ready its output is.

| Level | Meaning |
|-------|---------|
| **STABLE** | Actively maintained, production-ready, breaking changes follow semver. |
| **BETA** | Functional and community-supported, but the generated API may change between releases. |
| **EXPERIMENTAL** | Early-stage or proof-of-concept. Breaking changes can happen at any time. |
| **DEPRECATED** | Still present but no longer recommended. Will be removed in a future release. |

## Listing generators

```bash
# List all available generators
openapi-generator-cli list

# List only client generators
openapi-generator-cli list --include client

# List only server generators
openapi-generator-cli list --include server
```

## Getting help for a specific generator

```bash
# Show all config options for the java generator
openapi-generator-cli config-help -g java

# Output config options as markdown
openapi-generator-cli config-help -g typescript-axios -f markdown

# Output config options as YAML sample
openapi-generator-cli config-help -g typescript-axios -f yamlsample
```

## Server generators

Server generators produce framework scaffolding with handler stubs for each operation defined in your spec. You implement the business logic; the generator handles routing, parameter parsing, and response serialization.

  </Card>

  </Card>
  <Card title="Kotlin Spring" icon="code">
