OpenAPI Generator allows generation of API client libraries (SDK generation), server stubs, documentation, and configuration files automatically from an [OpenAPI Specification](https://github.com/OAI/OpenAPI-Specification). Both OpenAPI v2 (Swagger) and v3 (including 3.1 beta support) are supported.

## What is OpenAPI Generator?

Writing and maintaining API client libraries by hand is tedious, error-prone, and time-consuming. Every time your API changes, every SDK in every language needs to be updated. OpenAPI Generator solves this by automating the entire process — you define your API once in a standard OpenAPI spec, and the generator produces production-ready code across dozens of languages and frameworks.

The project is a community-driven fork of Swagger Codegen, with a focus on rapid iteration, quality, and broad language support. It is released under the Apache 2.0 license.

## Key capabilities

- **50+ languages and frameworks** for API clients, server stubs, and documentation
- **OpenAPI v2 and v3 support** — including Swagger 2.0 and OpenAPI 3.0/3.1
- **Multiple invocation methods** — CLI, Maven plugin, Gradle plugin, Docker, and an online API
- **Customizable templates** — override any part of the generated output using Mustache or Handlebars templates
- **Spec validation** — catch errors in your OpenAPI spec before generating code

## How to use OpenAPI Generator

You can invoke the generator in several ways depending on your workflow:

**CLI** — the most direct way to generate code from the command line. Available via npm, Homebrew, Scoop, PyPI, Docker, or a downloadable JAR.

**Maven plugin** — integrate generation into your Maven build lifecycle so code is always in sync with your spec.

**Gradle plugin** — add a Gradle task that generates code as part of your build.

**Docker** — run the generator without installing Java or any runtime dependencies.

**Online API** — use the hosted REST API or self-hosted Docker image to generate code on demand via HTTP.

<Warning>
  If your OpenAPI spec, templates, or any input is obtained from an untrusted source, review all inputs before running the generator to avoid potential code injection vulnerabilities. For security issues, contact [team@openapitools.org](mailto:team@openapitools.org).
</Warning>
